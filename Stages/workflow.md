# Workflow

Use this file to describe the overall workflow in human-readable terms.

## suggested structure

| Stage | Purpose | Depends On | Notes |
|---|---|---|---|
| `00-stage-template` | Reference template for creating new stages | - | Not an active stage unless explicitly used |

## guidance
- Summarize what each stage is for.
- Describe the rough order of work.
- Note important stage dependencies when they matter.
- Keep this high level; detailed stage guidance belongs in each stage's `intent.md`, `process.md`, and `contract.md`.
- Use `Stages/STATUS.json` for the live workflow state and explicit next steps.
